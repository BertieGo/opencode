# Subagent 深度解析 —— 项目经理与外包团队

> 用**公司项目管理**的比喻，带你彻底理解 OpenCode 的 Subagent 机制

---

## 🎭 场景设定：科技公司的一天

想象 OpenCode 是一家**科技公司**：

| 公司角色 | OpenCode 组件 | 职责 |
|---------|--------------|------|
| **CEO（客户）** | 用户 | 提出需求 |
| **项目经理** | Build Agent (Primary) | 总体规划、协调资源 |
| **外包专员** | Subagent (Task Tool) | 执行专项任务 |
| **外包公司类型** | Agent Types (explore/general) | 不同专业的外包团队 |
| **外包合同** | Task Tool Parameters | 任务描述和要求 |
| **工作间** | Sub Session | 独立的工作空间 |
| **工作报告** | Task Result | 外包团队返回的成果 |

```
┌─────────────────────────────────────────────────────────────────┐
│                      外包团队工作间                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   项目经理："explore 团队，请进你们的工作间"                      │
│        │                                                         │
│        ▼                                                         │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │   Session.create({                                       │   │
│   │     parentID: ctx.sessionID,  ← 记录父项目ID              │   │
│   │     title: "搜索权限相关代码 (@explore subagent)"         │   │
│   │     permission: [                                        │   │
│   │       { todowrite: "deny" },   ← ❌ 不能修改任务列表       │   │

│   │       { read: "allow" },       ← ✅ 可以读取文件           │   │
│   │       { bash: "allow" },       ← ✅ 可以执行命令           │   │
│   │     ]                                                   │   │
│   │   })                                                     │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│        │                                                         │
│        ▼                                                         │
│   🏢 独立工作间创建完成！                                         │
│      - 与主项目隔离                                               │
│      - 有自己的 session ID                                        │
│      - 权限严格受限                                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 关键技术细节

```
┌─────────────────────────────────────────────────────────────────┐
│                     主循环调用过程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   SessionPrompt.prompt({                  ← 外包团队的主循环    │
│     sessionID: session.id,                 ← 独立的工作间ID     │
│     agent: "explore",                      ← 使用 explore 配置  │
│     tools: {                               ← 受限的工具箱        │
│       todowrite: false,                                           │
│       todoread: false,                                            │
│       task: false,                     ← 关键：不能再创建子任务 │
│     },                                                            │
│     parts: promptParts,                ← 任务指令               │
│   })                                                              │
│                                                                  │
│   这相当于在子会话中运行一个完整的 "while(true)" 循环！           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 🎯 核心机制总结

### 调用机制

```
┌─────────────────────────────────────────────────────────────────┐
│                      Subagent 调用流程                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1. 主 Agent 调用 Task Tool                                     │
│      └── 填写 description, prompt, subagent_type                │
│                                                                  │
│   2. 权限检查                                                     │
│      └── 检查 caller 是否有权限调用该 subagent 类型             │
│                                                                  │
│   3. 创建独立 Session（子会话）                                   │
│      └── parentID 指向父会话                                     │
│      └── 独立的权限配置（通常更严格）                             │
│                                                                  │
│   4. 在子会话中启动主循环                                         │
│      └── SessionPrompt.prompt()                                 │
│      └── Subagent 自主运行多轮                                    │
│                                                                  │
│   5. 返回结果                                                     │
│      └── 包含 task_id（可用于续单）                              │
│      └── 包含 <task_result> 工作成果                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 关键特点

| 特点 | 说明 | 公司比喻 |
|------|------|---------|
| **隔离性** | 独立的 Session，独立的 Message 历史 | 独立的工作间 |
| **权限受限** | Subagent 通常权限更少 | 外包合同限制 |
| **防止套娃** | explore/general 默认不能再调用 task | 外包团队不能再外包 |
| **可续单** | 通过 task_id 可以恢复之前的子会话 | 项目编号，可以后续跟进 |
| **并行执行** | 可以同时调用多个 subagent | 同时雇佣多个外包团队 |
| **结果不直接可见** | 用户看不到原始 task result | 外包报告先给项目经理 |

### 预定义 Subagent 类型

```
┌─────────────────────────────────────────────────────────────────┐
│                     外包公司类型表                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   🔍 explore（代码探索专员）                                      │
│   ├── 权限：grep, glob, read, bash, websearch                   │
│   ├── 禁止：edit, write（只读）                                  │
│   ├── 禁止：task（不能再外包）                                    │
│   └── 用途：快速探索代码库，查找文件，搜索关键词                   │
│                                                                  │
│   🔬 general（通用研究员）                                        │
│   ├── 权限：与主 Agent 类似，但禁止 todowrite/todoread           │
│   ├── 禁止：task（不能再外包）                                    │
│   └── 用途：执行复杂的多步骤研究任务                               │
│                                                                  │
│   🏗️  build（主项目经理）                                         │
│   ├── 权限：完整权限，包括 task                                   │
│   └── 用途：主 Agent，可以调用其他 subagent                       │
│                                                                  │
│   📝 自定义 Agent                                                 │
│   └── 用户可以在 .opencode 配置中定义新的 subagent 类型           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 🍽️ 完整示例：@explore 命令的背后

```
用户输入：@explore 找出所有使用 useState 的组件

        │
        ▼
┌──────────────────────────────────────────────────────┐
│ 系统识别到 @explore 命令                               │
│ - 解析出 subagent_type: "explore"                     │
│ - bypassAgentCheck: true（用户明确指派，跳过权限检查）  │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────┐
│ 调用 Task Tool                                         │
│ {                                                      │
│   description: "Find useState usage",                  │
│   prompt: "找出所有使用 useState 的组件...",            │
│   subagent_type: "explore",                            │
│   command: "/explore"                                  │
│ }                                                      │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────┐
│ 创建新的 Sub Session（工作间）                          │
│ - title: "Find useState usage (@explore subagent)"    │
│ - permission: 只读权限（grep/glob/read）               │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────┐
│ explore 团队开始工作（主循环）                           │
│ 1. grep "useState" --glob="**/*.{tsx,jsx}"            │
│ 2. 读取几个典型文件                                    │
│ 3. 生成分析报告                                       │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────┐
│ 返回 Task Result                                       │
│ <task_result>                                          │
│ 找到 50 个使用 useState 的组件：                       │
│ - src/components/Button.tsx: 第 12 行...               │
│ - src/components/Modal.tsx: 第 8 行...                 │
│ ...                                                    │
│ </task_result>                                         │
└──────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────┐
│ 项目经理（Build Agent）总结并回复用户                    │
│ "我找到了 50 个使用 useState 的组件，主要分布在..."      │
└──────────────────────────────────────────────────────┘
```
